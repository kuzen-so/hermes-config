# 健康策略冲突案例：空腹玩游戏 → 头疼 → 归因错误

## 背景

用户同时追求三个目标：
1. 赚钱（核心，时间优先）
2. 减脂增肌（健康）
3. 形象（视频出镜需求）

执行策略："饿了才吃"（直觉饮食的简化版），用于减脂控制。

## 冲突表现

- 空腹玩游戏 → 头疼
- 头疼时喝咖啡（冰茉莉花香拿铁）缓解
- 归因于"游戏机制"或"需要咖啡因提神"
- 继续游戏，维持旧策略

## 归因错误拆解

| 变量 | 用户归因 | 实际贡献 |
|------|---------|---------|
| 停止玩游戏 | 忽略 | **主因**（解除视觉-前庭冲突） |
| 咖啡因 | 清醒感来源 | 次要，缩血管缓解头痛 |
| 牛奶/糖浆 | 未分析 | 血糖补充（如果空腹） |
| 空腹 | 未识别 | **根因**（低血糖+游戏负荷叠加） |

## 干预点

追问对照实验：
- "头疼时只停游戏，不喝任何东西，多久缓解？"
- "头疼时只喝白水，不停游戏，是否缓解？"

若停游戏 5-10 分钟缓解 → 游戏机制/姿势是主因，咖啡因是干扰项。
若必须喝咖啡才缓解 → 咖啡因戒断或疲劳是主因。

## 用户决策模式

用户通过"喝咖啡缓解"维持旧策略，延迟面对冲突。直到明确说"不能"接受当前代价，才进入策略调整对话。

## 解决方案

**减法主义整合：不增加新目标，合并执行。**

| 旧策略 | 新策略 | 理由 |
|--------|--------|------|
| 饿了才吃 | 起床后 2 小时内第一餐 | 定时定量，零决策消耗 |
| 空腹玩游戏 | 固定两餐后再玩 | 血糖稳定，不头疼 |
| 头疼喝咖啡 | 消除头疼来源 | 不处理症状，处理病因 |

**两餐固定菜单（极简）：**
- 第一餐 7:00：3全蛋3蛋白 + 1杯牛奶/豆浆
- 第二餐 14:00：200g鸡胸肉 + 干荞麦面60g + 蔬菜
- 每天一样，不轮换，不思考

## 关键教训

1. **目标叠加时，策略冲突比单一目标难识别。** 用户倾向于归因于表面症状（头疼），而非底层策略冲突（空腹+游戏）。
2. **"缓解"是维持坏策略的麻醉剂。** 咖啡让旧策略能继续跑，延迟了必须做的调整。
3. **减法主义执行：合并目标，减少决策。** 不是"减脂+增肌+形象+赚钱"四个目标，而是"赚钱+健康的身体"两个底座，形象是副产品。
4. **固定菜单 > 灵活选择。** 对减法主义用户，"每天一样"比"健康多样化"更易执行。
