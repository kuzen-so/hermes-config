# 健康策略冲突案例：空腹游戏头疼决策

**来源：** 2026-05-05 对话
**类型：** 决策日志 + 策略调整
**状态：** 已解决（策略淘汰）

---

## 背景

用户空腹玩游戏后出现头疼，通过喝含咖啡因饮料（库迪咖啡）缓解。用户同时有两个目标：
1. 减脂增肌 → 执行"饿了才吃"策略
2. 更多时间赚钱 → 需要持续高效产出

## 归因错误链

| 阶段 | 用户归因 | 实际主因 |
|------|----------|----------|
| 空腹玩游戏头疼 | "游戏导致头疼"或"需要咖啡提神" | 低血糖 + 视觉-前庭冲突 |
| 喝咖啡缓解 | "咖啡有效" | 咖啡因缩血管 + 停止玩游戏 |
| 维持旧策略 | "能跑通" | 延迟面对策略冲突 |

**关键混淆变量：** 停止玩游戏 vs 咖啡因，用户将缓解归因于后者。

## 干预对话

**追问策略：**
1. "头疼时有没有试过'停游戏 + 什么都不喝'？" → 拆出主因
2. "如果继续这个策略，头疼频率上升、咖啡因耐受增加、增肌停滞，你能接受吗？" → 迫使用户量化代价
3. 用户回答"不能" → 进入策略调整

**用户核心约束：**
- 不改"饿了才吃"规则本身（用户拒绝）
- 但接受"定时定量"替代"凭感觉饿"
- 增肌需要蛋白质定时摄入，与"饿了才吃"直接冲突

## 策略调整

**旧策略：** 饿了才吃（直觉饮食的简化版）
**新策略：** 定时两餐或三餐，每餐含蛋白质

| 维度 | 旧策略 | 新策略 |
|------|--------|--------|
| 触发 | 饿信号（已紊乱） | 时间点 |
| 决策消耗 | 每次判断"饿不饿" | 零（到点吃） |
| 增肌适配 | 差 | 好 |
| 头疼风险 | 高 | 低 |

## 关键判断

用户说"不能"接受代价时，才触发策略调整。此前所有缓解手段（咖啡、牛奶）都是维持旧策略的补丁。

**决策标准：**
- 能 → 继续，不干预
- 不能 → 提供替代方案，让用户选择两餐或三餐
