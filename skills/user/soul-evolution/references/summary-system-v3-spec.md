## 核心变更

从 v2.0 的单模板系统升级为 v3.0 的双路径系统。关键变化：

1. **触发条件不变**：用户说"总结"时触发，按主题切分
2. **新增"做了什么"定义**：区分输入（看了/听了/读了）和输出（写了/做了/决定了）
3. **新增"关闭状态"**：已解决（正向关闭/证伪/淘汰/合并）vs 未解决（仍在阻塞）
4. **双路径存放**：已解决→98-Logs/，未解决→01-To do/99-Question/
5. **双模板**：已解决模板聚焦"关键判断+策略"，未解决模板聚焦"卡在哪+下一步验证"
6. **冲突处理**：同一天同来源同问题，末尾加 `-01` 递增

→ 判定标准从"有没有突破"改为"做了什么"

→ 确认双模板结构

> "区分输入（看了/听了/读了）和输出（写了/做了/决定了）"

→ 输入/输出定义

> "关闭状态：已解决（正向关闭/证伪/淘汰/合并）/ 未解决（仍在阻塞）"

→ 四种关闭类型 + 一种未解决状态

> "阻塞类型：信息/能力/决策/外部"

## 与 v2.0 的差异对照

| 维度 | v2.0 | v3.0 |
|------|------|------|
| 判定标准 | 做了什么 | 做了什么 + 关闭状态 |
| 模板数 | 1 | 2 |
| 存放路径 | 只有 98-Logs/ | 98-Logs/ + 01-To do/99-Question/ |
| 策略定义 | 当前优先级最高的行动 | 此刻重来会优先做的决策（认知层面） |
| 冲突处理 | 无 | `-01` 递增 |
| 输入输出 | 不区分 | 强制区分 |
