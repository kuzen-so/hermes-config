# 日报时间线格式规范

## 背景
用户要求日报增加"时间线"作为第一层，替代碎片归纳功能。

## 格式

```markdown
## 日报

### 时间线
- HH:MM 平台 核心事件/判断
- HH:MM 平台 核心事件/判断

### 内容反思（What）
- 核心思考
- 关键行动
- 生活记录

### 过程反思（How）
- 认知突破
- 未完成疑问
- 能量分布

### 前提反思（Why）
- 前提质疑
- 身份投票

### 评价
- 具体行为评价，不贴标签
```

## 规则

1. **时间线只记转折点**，不记每句话
2. **平台标记**：微信/电脑/其他
3. **能看出思考过程**：哪个想法先出现，哪个是后来的修正
4. **跨平台统一排序**：按时间先后，不分平台
5. **评价必须具体**：杂乱、分散、验证不足、停在概念层
6. **禁止贴标签**：成长区/舒适区/恐慌区
7. **禁止泛泛赞美**

## 示例

```markdown
### 时间线
- 00:06 微信 开始 Soul 重构讨论，质疑双模式设计
- 01:36 微信 添加碎片归纳系统，每天 22:00 执行
- 11:52 微信 二燕家小孩抓周
- 16:10 电脑 开始优化 Soul，删除立赌/成长区判断
- 21:02 电脑 核心判断："生活是要做事，不是和自己较劲"
```

## 与碎片归纳的区别

| 维度 | 碎片归纳（已删除） | 日报时间线 |
|------|-------------------|-----------|
| 触发 | 22:00 自动 | 用户手动 |
| 粒度 | 按时段压缩（凌晨/上午/下午/晚上） | 按节点记录（HH:MM） |
| 内容 | 压缩成一段 | 列表形式 |
| 平台 | 不区分 | 标记平台 |
| 价值 | 看不出思考过程 | 能看出想法流转 |

## 历史变更

- v1.0：日报只有三层反思（What/How/Why）
- v2.0：增加时间线作为第一层，替代碎片归纳
- v2.1：删除碎片归纳 cronjob，日报完全手动触发
