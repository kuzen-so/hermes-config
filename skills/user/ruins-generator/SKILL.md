# Skill: Conversation Summary Generator

## 触发条件

用户说"总结"时触发。回顾当前对话上下文，按主题切分。

**一段对话 = 一件事 = 一篇。两件事 = 两篇。**

**拆篇信号（强制）：**
- 用户说"还有一件事""另外""换个话题"
- 上下文明显切换主题（从 A 改到 B）
- 用户明确说"这是另一件事"

不判定"有没有认知突破"，只判定"做了什么"。

## 执行纪律

**口头确认 ≠ 执行。** 当总结需要生成文件时，必须立即调用 write_file，不能只说"生成到 X"而不写。用户会纠正这个落差。

## 做了什么

区分输入（看了/听了/读了）和输出（写了/做了/决定了）。

## 关闭状态

- **已解决**：正向关闭 / 证伪 / 淘汰 / 合并
- **未解决**：仍在阻塞

## 路径

| 状态 | 存放位置 |
|---|---|
| 已解决 | `98-Logs/` |
| 未解决 | `01-To do/99-Question/` |

## 文件名

`YYYY-MM-DD-来源-核心问题.md`

**冲突处理**：同一天同来源同问题，末尾加 `-01` 递增。

示例：`2026-05-01-Hermes-input-output重构.md`

## 已解决模板

```markdown
# YYYY-MM-DD-来源-核心问题

## 做了什么
- [输入：导致行动的信息]
- [输出：实际产出的结果]

## 关键判断
[关闭类型：解决/证伪/淘汰/合并。一句话说明为什么到此为止。]

## 策略
[只有1条：此刻重来会优先做的决策。不是待办，是认知。]

## 关联日记
[[YYYY-MM-DD]]
```

## 未解决模板

```markdown
# YYYY-MM-DD-来源-核心问题

## 做了什么
- [输入：导致卡住的背景]
- [输出：实际尝试过的动作及结果]

## 卡在哪
[阻塞类型：信息/能力/决策/外部。一句话，最底层阻塞点。]

## 下一步验证
[只有1条。格式：做X，验证Y假设。]

## 关联日记
[[YYYY-MM-DD]]
```

---

## 规则

1. **时间**：只用绝对日期 `YYYY-MM-DD`。禁用今晚/昨天/明天。
2. **清单**：不列未来待办。只留此刻认知和下一步最小动作。
3. **字数**：全文不超过300字。超限强制拆篇。
4. **原创**：不存他人原句、不存AI生成原文。引用必须转述后删除原句。
5. **所有权**：只留自己的判断和 output。
6. **维护**：不整理、不维护、不设标签。写完即埋。

## 被我扔掉的改动（参考）

以下建议来自 CC，但我没有采纳，因为它们会重新引入维护负担：

- **来源详情**："对手是谁、原始议题是什么"——太细，容易变成 input 记录。来源写在文件名里足够。
- **被推翻/更新标记**：需要定期 review，维护成本高。错了就删，不用标记。
- **血统自检**：每次存文件都要做选择题 = 摩擦。所有总结默认是"在对话中被刺激后产出"。
- **风化期 30 天**：最大的维护陷阱。30 天后你不会去读的。如果写新文章时调用到旧总结发现失效，**当场删除**即可。
